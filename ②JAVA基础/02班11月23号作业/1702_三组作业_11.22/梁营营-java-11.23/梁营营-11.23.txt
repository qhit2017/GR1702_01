1.接口有什么好处？
  答：（1）接口中只能包含属性，方法；（2）接口可以实现多继承；（3）类可以继承接口，但是继承的类必须去
      实现接口中的方法代码；（4）接口不能直接被实例化，接口一般不用什么修饰符，因为接口默认的修饰符是
      Public公共的，接口中所有成员默认为public，因此接口中不能有pravate修饰符接口的好处：（5）对类的
      一种约束；（6）接口可以很容易进行拓展，并不影响以前写的功能模块的代码，只要新添加类，继承这个接
      口就可以了；（7）接口可以用于支持回调，而继承并不具备这个特点；（8）抽象类实现的具体方法默认为虚
      的，但实现接口的类中的接口方法却默认为非虚伪，当然您也可以声明为虚伪的。
2.总结抽象类与接口的区别：
  答：（1）接口不实现方法，但是可以调用接口的方法；（2）抽象类不能实例化；（3）接口的成员函数可以在不同
       的类中安装，而其他的成员函数可以有很多种实现方法，但是抽象类不行；（4）接口中的成员函数是预先定义
       出来的，而不用去实现；（5）接口可以多重继承，但是抽象类不能；（6）抽象类与之类是父子关系，而接口是
       松耦合的关系；（7）抽象类可以派生出一些成员的实现，但接口却不包含任何成员的实现；（8）接口可以有静
       态成员，虚拟成员，属性等。
